Problem24_Use_Case_Definition.md:
Overview
This task focuses on defining the core use cases of the Lost & Found Portal.
Use cases represent the main functionalities that allow actors to interact with the system.
Total Use Cases Defined
A total of 10 core use cases were identified to cover the complete lost-and-found workflow.
Core Use Cases
1.	Register & Login
2.	Report Lost Item
3.	Report Found Item
4.	Search Lost Items
5.	Search Found Items
6.	Submit Claim
7.	View Match Suggestions
8.	Manage Claims (Admin)
9.	Update Item Status
10.	Generate Reports
Purpose of Use Case Definition
•	To identify system functionalities clearly
•	To understand user interactions
•	To support further UML modeling
Outcome
The defined use cases provide a structured workflow for campus lost-and-found reporting, matching, and recovery operations.
