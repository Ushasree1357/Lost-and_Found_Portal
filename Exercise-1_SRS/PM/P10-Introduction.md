P10-Introduction:

1.1 Purpose
This Software Requirements Specification (SRS) document defines the functional and non-functional requirements for the Lost & Found Portal system.
1.2 Scope
The system provides a centralized online platform for university students and staff to report lost and found items, search for matching items, submit claims, and manage the recovery process, thereby improving campus efficiency and user convenience.
1.3 Definitions, Acronyms, and Abbreviations
•	LFP – Lost & Found Portal
•	SRS – Software Requirements Specification
•	UI – User Interface
•	Admin – System Administrator
•	User – Registered individual using the portal
